# 我的 AI 写作工作流：从提纲到发布

> 用 AI 辅助技术写作半年了，这篇文章总结了我摸索出来的一套工作流——不是让 AI 替你写，而是和 AI 协作写出更好的文章。

## 核心原则

用 AI 写作一年多，我总结了三条原则：

1. **AI 是副驾驶，不是代驾**——人负责方向、观点和判断
2. **AI 擅长广度，人擅长深度**——让 AI 覆盖信息面，人做深度思考
3. **迭代优于一次生成**——和 AI 多轮对话比一次长 prompt 效果好得多

## 我的五步工作流

```
提纲 → 素材 → 初稿 → 精修 → 发布
  ↑      ↑      ↑      ↑      ↑
 AI     AI     AI    人工   自动化
```

### 第一步：列提纲（AI 辅助）

对着 AI 描述想法，让它帮你展开成提纲：

> **我常这样开头**：
> "我想写一篇关于 [主题] 的文章，目标读者是 [人群]，帮我列一个详细的提纲，包含核心观点、论证结构和示例方向。"

AI 通常会给你一个不错的骨架，但你需要：
- 调整逻辑顺序（AI 有时会前后矛盾）
- 删掉太水的章节
- 补充你自己独特的经验视角

### 第二步：收集素材（AI 主力）

这是 AI 最擅长的环节。我会让 AI 帮我：

- **找代码示例**："给我 3 个 X 的实际代码示例"
- **查技术细节**："Y 功能的实现原理是什么？"
- **对比方案**："A 和 B 各有什么优缺点？"
- **找数据支撑**："有没有相关的统计数据或调查报告？"

关键技巧：对于不确定的信息，**要求 AI 标注信息来源**或自行验证。

### 第三步：生成初稿（人机协作）

这不意味着你把提纲扔给 AI，等着收文章。更好的方式是**分段协作**：

```text
# 错误方式
"帮我根据这个提纲写一篇 3000 字的文章"
# → 结果通常是 AI 风格的"水文"

# 正确方式
"请写第 2 节'XX'的内容，用技术博客的口吻，包含代码示例，约 500 字"
# → 分段控制质量，每段有明确方向
```

我通常的节奏是：
1. AI 写第 1 节 → 我审阅调整
2. AI 写第 2 节 → 参考第 1 节风格
3. 逐段迭代，保持风格一致

### 第四步：精修润色（人工为主）

**这 30% 的工序决定文章 70% 的质量。** 我主要做这几件事：

#### 去 "AI 味"

AI 生成的文字有一些典型特征，需要手动去掉：

| AI 味 | 改成人味 |
|--------|------|
| "总而言之/综上所述" | 直接说结论 |
| "值得注意的是/此外" | 省略或用"另外" |
| 每段开头都是"在…中" | 变换句式 |
| 罗列式分点 | 融入叙事 |
| 过度使用"非常/极其/显著" | 用数据说话 |

#### 加入真实经验

这是最重要的差异化要素。读者想读的是**你的真实经历**，不是 AI 的泛泛之谈：

```
❌ "配置过程通常比较顺利"
✅ "配置时我遇到一个坑：API Key 需要先在平台充值才能用，卡了我半小时"
```

#### 优化代码块

AI 生成的代码有时为了"看起来完整"而牺牲了可读性。我会：
- 删除无关的 import
- 补充关键注释
- 确保代码能直接运行
- 加上运行结果或截图

### 第五步：发布（自动化）

```bash
# 我的发布命令
git add posts/new-post.md
git commit -m "new post: XXX"
git push
# Vercel 自动部署，1 分钟后上线
```

发布后我会：
- 在社交媒体分享，附上文章最精彩的一段话
- 回复评论，文章才有生命力
- 根据反馈修改，博客文章是可以持续迭代的

|------|:---:|------|
| 教程/指南 | 40% | 代码示例、步骤展开 |

## 一些反思

### AI 写作的边界

AI 不太适合写：
- **强烈的个人观点**（AI 倾向中立客观）
- **一手实践经验**（AI 没有真正"用过"）
- **情感驱动的故事**（AI 的情感表达还是套路化的）

AI 非常适合：
- **教程和操作指南**（结构化的步骤说明）
- **技术对比评测**（多维度客观分析）
- **概念解释**（把复杂概念讲清楚）
- **代码示例生成**（多种写法对比）

### 关于"自己写"还是"让 AI 写"

有人担心用 AI 写作就"不是自己的作品了"。我的看法是：

> 锄头翻土、拖拉机也翻土，核心是土地上长出什么。写作的重点从来不是"每个字都是自己敲的"，而是文章传递的经验和思考是否有价值。

透明标注 AI 的参与程度，让读者自己判断，这是最好的做法。

## 下一步

如果你也想尝试 AI 辅助写作，建议这样开始：

1. 选一篇你之前纯手写的文章
2. 尝试用 AI 重新写提纲，对比结构
3. 让 AI 扩写其中一段，体会差异
4. 逐步建立你自己的协作节奏
